Getting started with your new BeagleBone

This is the quick-start guide. Please read the BeagleBone System Reference Manual and view the BeagleBoard101 presentation before looking for any additional help. It is also highly recommended for you to read the Frequently Asked Questions guide.

Before you begin, update to the newest demo image

The latest version of this Angstrom Cloud9 IDE demo image is available from the Angstrom Distribution BeagleBone demo page. Please visit there now to download the latest version. The BeagleBoard101 presentation will walk you through the process of writing the demo image to a microSD card if you aren't already a Linux user.

If you are using a different microSD card image, please look for support from the supplier of that image.

Some software already running on your BeagleBone

The Angstrom Distribution image provided with your BeagleBone includes a large amount of software already running as you are reading this document. It is also possible to install many more pre-built packages from the on-line repository feed using the 'opkg' tool.

Some of the running programs include web servers and other Internet servers that give you access to the board, even if you don't have a display, mouse and keyboard connected to it.

Here is a list of some of the running servers:

  • ssh (port 22) -
  • boneserver (port 80) -
  • GateOne (port 443) -
  • Cloud9 IDE (port 3000) -

Decision time: Ethernet or single-cable USB based development?

The most recommended path from here for someone experienced in managing their local network and performing remote logins via ssh is to go get a recommended 5V power adapter and connect your BeagleBone via an Ethernet to your LAN. Your BeagleBone will automatically start web services to which you can connect, so you can avoid installing USB drivers until you want to perform some lower-level debug. Use Bonjour or your router's DHCP log to discover the IP address of your board and browse to it.

Further, right now, the clock speed of the BeagleBone is determined by if the power source is USB or the 5V adapter at initial boot. It will be 500MHz if the power source is USB and 720MHz if the power source is an adapter.

Getting your host machine drivers setup

This BeagleBone demo images exposes five (5) different USB device types to your host machine:

  1. SMSC USB hub
  2. Linux mass storage class interface
  3. FTDI USB-to-serial interface
  4. FTDI USB-to-JTAG interface
  5. Linux USB-to-Ethernet interface

Neither of the SMSC USB hub and Linux mass storage class interface will require any driver installation on your host machine as drivers for those are completely standard in all major operating systems. Your BeagleBone comes with a built-in USB-to-serial interface adapter. To enable it, you'll need to install the Windows, Mac OSX or Linux driver.

  • Windows:
    • Code Composer Studio v5 (CCSv5) is a TI provided Eclipse-based integrated development environment that enables you to debug and develop C/C++ applications for your BeagleBone. If you've already installed CCSv5, you already have the driver and you can skip this step. CCSv5 can be used with the BeagleBone for free. See the CCSv5 wiki page for download and installation information.
    • Otherwise, point your Windows machine to the \Drivers\Windows\FTDI folder for the drivers. If you closed the window, double-click each of the .inf files to manually install the drivers.
    • Start Hyperterminal and point it to the newest COM port.
  • Mac OS X:
    • Open /Drivers/MacOSX/FTDI/FTDI_Ser.dmg, double-click "FTDI_Serial.pkg" and follow the instructions. For additional information, visit ftdichip.com.
    • To open the serial terminal from within Terminal.app, use the following command: 
      screen /dev/tty.usb*B 115200
  • Linux:
    • For a quick start, simply use the following commands: 
      sudo modprobe ftdi_sio vendor=0x0403 product=0xa6d0
screen /dev/`dmesg  | grep FTDI | grep "now attached to" | tail -n 1 | awk '{ print $NF }'` 115200
    • To make the BeagleBone show up upon every reboot, add a udev rule. Here is an example /etc/udev/rules.d/73-beaglebone.rules: 
      ACTION=="add", SUBSYSTEM=="usb", ENV{DEVTYPE}=="usb_interface", \
        ATTRS{idVendor}=="0403", ATTRS{idProduct}=="a6d0", \
        DRIVER=="", RUN+="/sbin/modprobe -b ftdi_sio"

ACTION=="add", SUBSYSTEM=="drivers", \
        ENV{DEVPATH}=="/bus/usb-serial/drivers/ftdi_sio", \
        ATTR{new_id}="0403 a6d0"

ACTION=="add", KERNEL=="ttyUSB*", \ ATTRS{interface}=="BeagleBone", \
        ATTRS{bInterfaceNumber}=="00", \ SYMLINK+="beaglebone-jtag"

ACTION=="add", KERNEL=="ttyUSB*", \ ATTRS{interface}=="BeagleBone", \
        ATTRS{bInterfaceNumber}=="01", \ SYMLINK+="beaglebone-serial"

Step #2: Install the USB-to-Ethernet device driver

The Linux distribution running on your BeagleBone supports emulating a USB-to-Ethernet device such that you can access your BeagleBone with networking protocols, such as web browsing.

  1. Prepare for the driver installation.
    • Windows:
      • Copy linux.inf from the "\Drivers\Windows\RNDIS" folder to your PC for use to tell Windows where to find the driver.
    • Mac OS X:
      • The folks over at Bug Labs have produced a driver you might use: http://bugcommunity.com/wiki/index.php/Start_Guide_Mac_OS_X_10.5
    • Linux:
      • No driver installation should be required, simply configure the adapter for DHCP.
  2. Eject the BEAGLE_BONE USB drive. The USB Ethernet gadget will automatically start.
  3. Configure the adapter that appears for DHCP.

Step #3: Navigate to your BeagleBone's web interface

The Linux distribution running on your BeagleBone is configured to start a handful of web servers upon boot. This is the easiest way to get explore and program your BeagleBone if you are not already experienced with Linux.

Please note that many of the tools do not support Internet Explorer. It is recommended you use recent versions of Chrome or Firefox.

  • For an alpha board overview status app, visit http://192.168.7.2.
  • For Matrix, visit http://192.168.7.2:8080/index.html.
  • For Cloud9 IDE, visit http://192.168.7.2:3000.
